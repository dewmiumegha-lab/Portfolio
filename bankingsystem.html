Java Banking System

A desktop banking management application with a Swing GUI. Supports real-time deposit, withdrawal, and transfer operations with CSV-based persistent data loading.

Java Java Swing GUI OOP CSV File I/O Desktop App

📋 Project Overview

The Java Banking System is a desktop application built with Java Swing that simulates core banking operations. Account data is loaded from a CSV file at startup, and all changes — deposits, withdrawals, transfers — are reflected live in the GUI table. The project demonstrates strong OOP principles including inheritance, encapsulation, and abstraction.

📄 CSV Data Format

Account data is stored in Accounts.csv with no header row. Each line represents one account: FirstName, LastName, AccountNumber, Balance.

# Accounts.csv — no header row # Format: FirstName, LastName, AccountNumber, Balance Dewmi,Umegha,11,10000 Udara,Nethmi,22,10 Ginuka,Weragoda,33,50000 Sanduni,Priyalakshi,44,99999

At startup, ReadAccounts parses this file into four LinkedList collections, which are then used to build the Account[] array passed to the GUI.

User interactions in the GUI trigger ActionListener events via an inner HandlerClass, which calls the appropriate methods on the relevant Account or Transaction objects, then refreshes the JTable model.

🎓 OOP Concepts Applied

🧬 Inheritance

Account extends Customer to reuse name fields and accessor methods without code duplication.

🔒 Encapsulation

All fields are declared private. Public getters and setters control access, protecting data integrity.

🎭 Abstraction

The Transaction class hides transfer logic from the GUI layer — the GUI just calls transfer() without knowing the internals.

⚡ Event-Driven

Inner HandlerClass implements ActionListener for all buttons, keeping event logic close to the UI it serves.

🚀 Possible Improvements

  • Write updated balances back to Accounts.csv after each transaction
  • Add overdraft protection — check balance before allowing a withdrawal
  • Refactor ReadAccounts to parse CSV in a single pass instead of four separate passes
  • Add account creation and deletion directly from the GUI
  • Replace CSV storage with a SQLite or MySQL database for better scalability
  • Add a transaction history log displayed in a separate scrollable panel
