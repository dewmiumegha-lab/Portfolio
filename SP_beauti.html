🔍 SQL Queries

Seven queries demonstrating key SQL features — from basic filtering to joins, aggregation, and subqueries.

QUERY 1 — BETWEEN
Treatments Priced Rs. 1,000 – 4,000
Retrieves treatments within a specific price range using the BETWEEN operator.
SELECT treatment_id, treatment_name, price FROM Treatment WHERE price BETWEEN 1000 AND 4000; TREATMENT_ID TREATMENT_NAME PRICE 301 Deep Clean Facial 1000.00 302 Swedish Massage 1500.00 303 Hot Stone Massage 2000.00 304 Luxury Manicure 3000.00 307 Makeup Session 3000.00 5 rows selected.
QUERY 2 — DISTINCT
Unique Staff Specialisations
Returns each distinct specialisation offered by the spa's staff, eliminating duplicates.

QUERY 3 — TO_CHAR
Formatted Appointment Dates
Uses TO_CHAR to display appointment dates in readable DD-MON-YYYY format.

QUERY 4 — AGGREGATE (SUM)
Total Revenue from Paid Payments
Uses SUM() to calculate total confirmed revenue, filtering only 'Paid' records.
SELECT SUM(amount) AS total_revenue FROM Payment WHERE payment_status = 'Paid'; TOTAL_REVENUE 17645 1 row selected.
QUERY 5 — GROUP BY
Appointments per Staff Member
Groups appointments by staff_id and counts each staff member's total bookings.

Premium Customers (Treatments > Rs. 3,000)
Uses a correlated subquery with JOIN to find customers who booked high-value treatments.

✏️ DELETE, UPDATE & INSERT Operations

Each DML type was tested with both a successful and a failed case to demonstrate Oracle's foreign key integrity enforcement.

✅ DELETE — Success
DELETE FROM Payment WHERE payment_id = 508
Payment row has no child references — deleted cleanly.
❌ DELETE — Failed
DELETE FROM Appointment WHERE appointment_id = 401
ORA-02292: Payment table has a matching FK row — delete blocked.
✅ UPDATE — Success
UPDATE Customer SET phone = '0700000000' WHERE customer_id = 101
Phone has no FK relationship — updated without issue.
❌ UPDATE — Failed
UPDATE Appointment SET appointment_id = 999 WHERE appointment_id = 401
ORA-02292: Payment references this PK — changing it breaks the FK chain.
✅ INSERT — Success
INSERT INTO Staff VALUES (209, 'Sandali', 'Nethmini', 'Therapist', 'Massage')
Staff has no FK dependency — row created successfully.
❌ INSERT — Failed
INSERT INTO Appointment VALUES (500, 999, 201, 301, ...)
ORA-02291: customer_id = 999 does not exist in Customer — parent key not found.

🔐 Multi-User Security Setup

Two Oracle users were created with different privilege levels to demonstrate role-based access control.

User Password Privileges
c##2435430 boss_pass CREATE SESSION CREATE TABLE (+ ADMIN) SELECT INSERT UPDATE DELETE
c##2406982 data_pass CREATE SESSION CREATE TABLE SELECT (+ GRANT) INSERT UPDATE DELETE
-- Create users CREATE USER c##2406982 IDENTIFIED BY data_pass; CREATE USER c##2435430 IDENTIFIED BY boss_pass; -- Grant login GRANT CREATE SESSION TO c##2406982, c##2435430; -- Grant table creation (c##2435430 can pass it on) GRANT CREATE TABLE TO c##2435430 WITH ADMIN OPTION; GRANT CREATE TABLE TO c##2406982; -- Grant object privileges on Customer GRANT SELECT ON Customer TO c##2406982 WITH GRANT OPTION; GRANT SELECT, INSERT, UPDATE, DELETE ON Customer TO c##2435430;
