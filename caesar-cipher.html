📋 Project Overview

The Caesar Cipher is one of the simplest and most well-known encryption techniques. It works by shifting each letter of a message by a fixed number of positions in the alphabet. This Python project implements both encryption and decryption through a command-line interface.

🔑 How Caesar Cipher Works

Each letter in the plaintext is shifted by a fixed number of positions (the "key") down the alphabet. For example, with a shift of 3:

Plaintext
HELLO
→ shift 3 →
Ciphertext
KHOOR

To decrypt, you simply shift in the reverse direction by the same number.

⚙️ Features

  • Encrypt any text message using a user-defined shift key
  • Decrypt any ciphertext back to plaintext
  • Handles both uppercase and lowercase letters
  • Non-alphabetical characters (spaces, punctuation, numbers) are preserved unchanged
  • Input validation and user-friendly error messages
  • Loop to allow multiple encryptions/decryptions without restarting

🛠️ Core Code Logic

The core logic uses Python's ord() and chr() functions to shift characters within the ASCII range of the alphabet.

def caesar_cipher(text, shift, mode): result = "" if mode == "decrypt": shift = -shift # Reverse the shift for decryption for char in text: if char.isalpha(): base = ord('A') if char.isupper() else ord('a') result += chr((ord(char) - base + shift) % 26 + base) else: result += char # Keep spaces, numbers, punctuation return result

💡 Challenges & Learnings

The trickiest part was handling the wrap-around — for example, ensuring that shifting ‘Z’ correctly returns to ‘C’ instead of going outside the alphabet required careful thinking. I solved this efficiently by using the modulo operator (% 26), which keeps values within the range of 26 letters.

Through this project, I strengthened my understanding of ASCII values, string manipulation, and writing reusable functions in Python. Additionally, it gave me a basic introduction to symmetric-key cryptography, where the same key is used for both encryption and decryption.
