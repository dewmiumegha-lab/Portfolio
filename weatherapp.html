Weather App

A real-time weather application built with HTML, CSS, and JavaScript using a public weather API.

HTML CSS JavaScript REST API Responsive Design

📋 Project Overview

This Weather App allows users to search for any city and instantly view current weather conditions including temperature, humidity, wind speed, and weather description. The app fetches live data from a public weather API and displays it in a clean, user-friendly interface.

🎯 Problem Statement

The goal of this project was to create a useful web app that shows how to get data from external APIs. It also demonstrates how to use JavaScript to handle tasks in the background and display live weather information in a clean, professional design. Building a weather app is a perfect way to practice these essential front-end development skills.

⚙️ Features

  • Search any city worldwide to get current weather
  • Displays temperature (°C / °F), humidity, and wind speed
  • Dynamic weather icons that change based on conditions (sunny, rainy, cloudy, etc.)
  • Error handling for invalid city names

🛠️ Tech Stack & How It Works

The app is built with plain HTML, CSS, and JavaScript — no frameworks needed. It calls the OpenWeatherMap API to fetch weather data based on the user's search input.

// Fetch weather data from OpenWeatherMap API async function getWeather(city) { const url = `https://api.openweathermap.org/data/2.5/weather?q=${city}&appid=${API_KEY}&units=metric`; const response = await fetch(url); const data = await response.json(); displayWeather(data); }
  • HTML — Page structure and layout
  • CSS — Styling, animations, and responsive grid
  • JavaScript (async/await) — API calls, DOM manipulation, error handling
  • OpenWeatherMap API — Live weather data source

💡 Challenges & Learnings

One of the main challenges was handling API errors gracefully — for example, when a user types an incorrect city name. I implemented try/catch blocks and conditional checks on the API response status code to show user-friendly error messages instead of crashing the app.

I also learned how to work with asynchronous JavaScript using async/await and how to dynamically update the DOM based on API response data. This project significantly improved my understanding of how real-world web apps consume external services.
