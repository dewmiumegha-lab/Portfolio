Cinema Booking System

A full-stack Java application combining a console interface, MySQL database, and a RESTful web service — supporting complete CRUD operations for movies, customers, screenings, and bookings.

Java MySQL JDBC REST Web Service OOP / SOLID JSON Postman

📋 Project Overview

The Cinema Booking System — "Grand City Cinema" — is a two-part Java project. Part 1 is a console-based CRUD application connected to a MySQL database via JDBC. Part 2 exposes the same logic as a RESTful web service using Java's built-in com.sun.net.httpserver, testable via Postman. Both parts share the same OOP model layer.

🌐 REST Web Service Endpoints

CinemaWebService.java starts an HTTP server on port 8080. All responses are in JSON format with CORS enabled.

GET http://localhost:8080/movies
POST http://localhost:8080/bookings

Each handler is implemented as a static inner class implementing HttpHandler. Requests are parsed from JSON using the org.json library, and responses include appropriate HTTP status codes (200, 201, 404, 405).

🎓 OOP & SOLID Principles Applied

🧬 Inheritance

Customer extends User, reusing id/name/email fields and adding membership-specific behavior without duplication.

🔒 Encapsulation

All fields across every class are private (or protected in User). Data is accessed only through getters/setters with validation (e.g., price and duration cannot be negative).

🎭 Polymorphism

Customer.displayInfo() overrides User.displayInfo(). The Payable interface allows Booking to be treated polymorphically as a payable object.

🧱 Composition

Screening contains a List<Seat> — seats are created and managed by the screening, showing a "has-a" composition relationship.

📐 SRP

Each class has one clear responsibility: Movie handles movie data, Seat handles seat state, Booking handles booking + payment.

🔌 ISP

The Payable interface is focused and minimal — only processPayment() and generateReceipt() — not a bloated general-purpose interface.

🚀 Possible Improvements

  • Add PUT endpoint to the REST service for updating movie prices and customer info
  • Build a simple HTML/JS front-end to consume the REST API instead of Postman
  • Add JWT-based authentication to protect the booking and customer endpoints
  • Implement pagination for large movie and booking lists
